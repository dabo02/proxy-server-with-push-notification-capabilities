Requests:

1) check for not allowed parametters on request URI, if there are remove them.
2) replace request URI with mercurio proxy's URI
3) check for max-forwards header field on requests URI's field values, if there is not one add it.
4) * insert record route header field values (its optional but reasures reconection on failure event)
5) if 4) then the inserted values must contain an lr(loose route) parameter (Section 19.1.1). (headers will be SIP URI and transport).
6) * add aditional header fields if needed.
7) Determine next-hop address port and transport
8) Add Via header
9) Add content-length header
10) create new client transaction for request(Section 17.1) using address port and transport determined on 7)
11) set timer C described in Section 16.7 bullet 2 of rfc spec.
